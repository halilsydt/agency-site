# Story 4.8: Production Launch Checklist

## Status

Approved

## Story

**As a** developer,
**I want** all launch requirements verified,
**so that** the site is ready for public announcement.

## Acceptance Criteria

1. Custom domain configured and working
2. SSL certificate valid
3. All placeholder content replaced with real content
4. All links working (no 404s)
5. Forms tested and receiving submissions
6. Booking integration working
7. Analytics receiving data
8. Social links correct
9. Legal pages published
10. Favicon and social images uploaded
11. Site submitted to Google Search Console
12. Backup/recovery plan documented

## Tasks / Subtasks

- [ ] Task 1: Domain and SSL Verification (AC: 1, 2)
  - [ ] Verify custom domain is configured in Vercel project settings
  - [ ] Confirm DNS records are properly configured (A, CNAME, or nameservers)
  - [ ] Test that domain resolves correctly to the site
  - [ ] Verify SSL certificate is valid and not expiring soon
  - [ ] Test HTTPS redirect is working (http:// → https://)
  - [ ] Check for mixed content warnings in browser console

- [ ] Task 2: Content Audit (AC: 3)
  - [ ] Review all JSON content files in `content/` directory for placeholder text
  - [ ] Check `content/services/amazon.json` and `content/services/etsy.json` for real service descriptions
  - [ ] Verify `content/pricing/packages.json` has final pricing (or approved placeholder)
  - [ ] Review `content/testimonials.json` for real or approved placeholder testimonials
  - [ ] Check `content/faqs.json` for complete FAQ content
  - [ ] Verify `content/team.json` has real team information
  - [ ] Check `content/results.json` for dashboard screenshots/results
  - [ ] Review `content/site-config.json` for correct business information
  - [ ] Scan all page components for hardcoded placeholder text ("Lorem ipsum", "TODO", etc.)

- [ ] Task 3: Link Verification (AC: 4)
  - [ ] Run link checker tool across the entire site (use `npx broken-link-checker` or similar)
  - [ ] Manually verify all navigation links (header, footer, mobile menu)
  - [ ] Test all internal page links
  - [ ] Test all CTA buttons link to correct destinations
  - [ ] Verify service cards link to correct service pages
  - [ ] Test pricing page CTAs link to contact/booking
  - [ ] Verify footer quick links all work
  - [ ] Check for any 404 errors in browser console during navigation

- [ ] Task 4: Form Testing (AC: 5)
  - [ ] Test contact form submission on `/contact` page
    - [ ] Submit with valid data and verify Formspree receives the submission
    - [ ] Verify success message displays correctly
    - [ ] Test form validation (required fields, email format)
    - [ ] Verify error handling for failed submissions
  - [ ] Test email signup form in footer
    - [ ] Submit with valid email and verify ConvertKit receives the subscription
    - [ ] Verify success message displays correctly
    - [ ] Test validation for invalid email format

- [ ] Task 5: Booking Integration Testing (AC: 6)
  - [ ] Navigate to `/contact` page
  - [ ] Verify Cal.com embed loads correctly
  - [ ] Test that calendar shows available time slots
  - [ ] Attempt to book a test appointment (can cancel immediately)
  - [ ] Verify booking confirmation works correctly
  - [ ] Test on mobile device for responsive booking experience

- [ ] Task 6: Analytics Verification (AC: 7)
  - [ ] Log into Plausible dashboard
  - [ ] Navigate through all pages on the live site
  - [ ] Verify pageviews are being recorded in Plausible
  - [ ] Test event tracking:
    - [ ] Submit contact form and check for event
    - [ ] Submit email signup and check for event
    - [ ] Click booking CTA and check for event
  - [ ] Verify analytics respects cookie consent settings

- [ ] Task 7: Social Links Verification (AC: 8)
  - [ ] Locate all social links in footer
  - [ ] Click and verify Instagram link opens correct profile
  - [ ] Click and verify LinkedIn link opens correct page
  - [ ] Click and verify Twitter/X link opens correct profile (if applicable)
  - [ ] Verify social links open in new tab (`target="_blank"`)
  - [ ] Verify social links have proper `rel="noopener noreferrer"` for security

- [ ] Task 8: Legal Pages Verification (AC: 9)
  - [ ] Navigate to `/privacy` and verify content is published (not placeholder)
  - [ ] Navigate to `/terms` and verify content is published (not placeholder)
  - [ ] Verify "Last Updated" date is recent and accurate
  - [ ] Check Privacy Policy mentions: Plausible, Formspree, ConvertKit, Cal.com
  - [ ] Verify footer links to Privacy Policy and Terms work
  - [ ] Verify cookie consent banner links to Privacy Policy

- [ ] Task 9: Favicon and Social Images (AC: 10)
  - [ ] Verify favicon displays correctly in browser tab
  - [ ] Test favicon on multiple browsers (Chrome, Firefox, Safari)
  - [ ] Verify Open Graph image is configured in metadata
  - [ ] Test social sharing preview:
    - [ ] Use Facebook Sharing Debugger to test OG tags
    - [ ] Use Twitter Card Validator to test Twitter cards
    - [ ] Verify image, title, and description appear correctly
  - [ ] Check for apple-touch-icon in public folder (for iOS bookmarks)

- [ ] Task 10: Google Search Console Submission (AC: 11)
  - [ ] Access Google Search Console
  - [ ] Add and verify property for the custom domain
  - [ ] Submit `sitemap.xml` URL to Search Console
  - [ ] Request indexing for homepage
  - [ ] Verify no crawl errors or issues are reported
  - [ ] Document Search Console access credentials/account used

- [ ] Task 11: Backup and Recovery Documentation (AC: 12)
  - [ ] Document current deployment configuration
  - [ ] List all environment variables and their purposes (without values)
  - [ ] Document Vercel project settings that were configured
  - [ ] Create rollback procedure:
    - [ ] How to deploy previous version via Vercel dashboard
    - [ ] How to revert Git commits if needed
  - [ ] Document third-party service credentials locations:
    - [ ] Formspree account access
    - [ ] ConvertKit account access
    - [ ] Cal.com account access
    - [ ] Plausible account access
    - [ ] Google Search Console access
  - [ ] Create simple README section or separate doc for disaster recovery

- [ ] Task 12: Final Verification Sweep (AC: 1-12)
  - [ ] Run `npm run build` - verify successful build with no errors
  - [ ] Run `npm run lint` - verify no lint errors
  - [ ] Run `npm run test:run` - verify all tests pass
  - [ ] Run Lighthouse audit on production URL:
    - [ ] Performance score > 90
    - [ ] Accessibility score > 90
    - [ ] Best Practices score > 90
    - [ ] SEO score > 90
  - [ ] Document all verification results
  - [ ] Create launch sign-off checklist summary
